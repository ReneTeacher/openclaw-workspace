## 執行摘要

Eng-Test-Maker 係一個已有完整 AI 自動化流程嘅英文科考試系統，已包含聽寫自動朗讀、影相上傳 OCR、AI 答案批改、以及 Email 成績報告四大核心功能。系統已部署上 Render，成本結構清晰。用戶係澳門中學英文老師，目標係建立一個可持續收費嘅 SaaS 產品。

### 4.2 API 使用現狀分析

**你目前使用三個主要 API：**

#### A. MiniMax API（abab6.5s-chat）
- **用途**：AI 句子分割、語意比對、詞彙評分
- **成本**：abab6.5s-chat 輸出 tokens 約 $0.015/千tokens（估算）
- **評估**：✅ server-side，冇問題
- **每月估算（100 個學生 submission）**：$1-4

#### B. Poe API（Claude/GPT）
- **用途**：OCR 圖片理解 + 句子 matching
- **成本**：Poe 按用量收費，Claude/GPT 3.1 Flash 便宜
- **評估**：✅ server-side，走 Poe proxy 係聰明做法
- **每月估算（100 個學生 submission）**：$0.2-1

#### C. SMTP Email（Nodemailer）
- **用途**：成績報告 Email 發送
- **問題**：SMTP 設定複雜，容易被 Gmail block
- **評估**：❌ 建議更換

## 第五部分：澳門商業化策略

### 5.1 支付解決方案（最重要問題）

**問題：Stripe 唔支援澳門，點算？**

**方案一：Stripe HK（推薦）**
- 澳門居民可以用香港身份證申請 Stripe HK
- 或者成立一間香港公司（成本約 HK$3,000-5,000）
- Stripe HK 支援信用卡全球收款
- **難度**：中等，需要搞香港公司

**方案二：LemonSqueezy（最簡單）**
- 申請容易，唔需要香港公司
- 內建全球 VAT 處理
- 可以嵌入 landing page
- **缺點**：澳門用戶支援未完全確認
- **難度**：低

**方案三：PayPal（最保守）**
- 澳門 PayPal 支援良好
- 用戶基礎大，但澳門人少用
- **缺點**：手續費高，用户体验一般
- **難度**：低

**方案四：FPS 直接轉帳（本地化）**
- 澳門朋友可以直接 FPS 轉帳俾你
- 適用於熟人市場
- **缺點**：無法自動化，唔適合 SaaS 規模化
- **難度**：極低

**建議**：先用 LemonSqueezy 或 PayPal，之後再做 Stripe HK

### 5.2 定價策略

**推薦定價結構（澳門/香港市場）**

| 計劃 | 價格 | 包含 |
|------|------|------|
| **Free** | $0 | 每月 10 個學生 submission |
| **Basic（月付）** | $10/月 | 每月 100 個學生 submission |
| **Pro（月付）** | $20/月 | 每月 500 個學生 submission |
| **學校授權** | $100/月 | 學校內所有老師，無限 submission |

**定價邏輯：**
- $10/月 vs 科大訊飛 HK$50,000/年——性價比極高
- 相對於老師自己時間價值——每月慳幾小時批改
- 澳門家長願意為教育付費

### 5.3 分銷策略——如何利用你的關係網絡

**你朋友的价值：**
- 朋友老婆的契媽係濠江英才校長陳紅
- 呢個係極高價值的關係入口
- **唔需要靠科大訊飛，靠呢層關係直接sell你嘅product**

**建議做法：**

**第一步：借濠江英才做標杆案例**
- 搵朋友介紹，俾陳紅或英文科主任免費試用
- 做出成績之後，請陳紅寫推荐信/介紹其他學校
- 重點：唔好話係想卖嘢，係話想幫澳門教育

**第二步：設計分銷佣金（避免50-60%問題）**
- 建議佣金的30-40%（非50-60%）
- 佣金的基礎係「成功介紹學校/老師」
- 唔係「每個月營業額」
- 設定期限（例：介紹費只俾12個月，其後自行續約）

**第三步：用標杆案例進入社群**
- 澳門老師社群細，容易靠口碑相傳
- Facebook/WhatsApp 教師群組係主要渠道
- 第一個付費用戶示範效應極强

## 第九部分：你問的所有問題解答

### Q1：什麼係 SaaS？
**A：** SaaS = Software as a Service，軟件即服務。你唔係卖軟件，係讓老師月費訂閱。似 Netflix / Spotify 模式。

### Q2：我的 API 用法點評？
**A：**
- MiniMax ✅（server-side，冇問題）
- Poe ✅（server-side，聰明做法）
- SMTP ❌（太複雜，轉去 Resend）

### Q3：每個月 API 成本幾多？
**A：** $1.2-5/月（100個學生）。收 $15/月，毛利 $10-13 ✅

### Q4：用家完全唔需要 set API？
**A：** ✅ 正確，你嘅架構已經係 All server-side。用家只接觸 UI。

### Q5：點樣做到訂閱系統？
**A：** 用 LemonSqueezy（最簡單）或 Stripe HK（長遠目標）。

### Q6：每個月 API 係咪我自己出？
**A：** 係，你每個月 API 成本約 $1-5，但你可以收 $15+/月，有利潤。

### Q7：點解 50-60% 俾朋友係貴？
**A：** 如果你每月收 $15，俾朋友 $9，你得 $6。如果100個用户，你每月只袋 $600，但朋友袋 $900。建議降到 30-40%，並設定期限。

### Q8：澳門 Stripe 問題點解決？
**A：** 用 LemonSqueezy 或 PayPal 作為過渡，長遠做 Stripe HK。
